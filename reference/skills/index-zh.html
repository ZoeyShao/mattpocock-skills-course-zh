Skill 中文阅读目录

每一项都链接到 mattpocock/skills 仓库中非 deprecated 的 SKILL.md HTML 渲染版。左边保留原始 skill 名称，右边是中文用途说明。遇到不熟的工程词，先查 术语解释。

路由与准备

Skill中文用途
ask-matt当你不确定该走哪条 skill flow 时，用它当路由器。
setup-matt-pocock-skills给 repo 写入 issue tracker、triage 标签和领域文档约定。

发现、需求与领域语言

Skill中文用途
decision-mapping把模糊想法拆成可逐个解决的研究、讨论或原型票据。
grilling对一个计划或设计进行一问一答的高强度追问。
grill-me无代码库时，用追问方式把计划或设计问清楚。
grill-with-docs一边追问计划，一边对照并更新 CONTEXT.md 和 ADR。
prototype用一次性 UI 或逻辑原型回答设计问题。
to-prd把已有上下文整理成 PRD，并发布到 issue tracker。
domain-modeling澄清领域术语、更新领域模型，并在必要时记录 ADR。

架构、切片与实现

Skill中文用途
codebase-design提供 deep module、interface、seam、adapter 等架构设计词汇。
improve-codebase-architecture扫描代码库，找出可以加深模块、改善可测性的架构机会。
to-issues把 PRD 或计划拆成可独立完成的纵向切片 issues。
triage处理 issue tracker 里的原始问题，分类并推进状态。
implement根据 PRD 或 issue 实现代码，并最终 review、提交。
tdd用红灯、绿灯、重构循环实现功能或修 bug。
diagnosing-bugs用反馈循环、复现、假设、探针和回归测试诊断困难 bug。
migrate-to-shoehorn把测试里的 TypeScript as 断言迁移到 @total-typescript/shoehorn。
scaffold-exercises按课程结构生成 exercises 目录、题目、解答和 explainer。
