切片与 Triage

第 0007 课 · 工程周期第 4 阶段 · 15 分钟

这个阶段把“文档里的计划”变成“可以被不同人或 agent 抓取的工作”。

to-issues：纵向切片

这里的 issue 不是只指 bug。它更像“工作票据”或“任务卡”：可以是 bug、功能、重构、QA 反馈，也可以是从 PRD 拆出来的一小块实现工作。

to-issues 把 PRD 或计划拆成 tracer bullet issues。每个 issue 应该穿过 schema、API、UI、测试等必要层，形成一条窄但完整、可验证的路径。

反例是横向切片：一个 issue 只建数据库，一个 issue 只写 API，一个 issue 只写 UI。这样每个 issue 都不可独立验证。

triage 的位置

  • triage：处理已经进入 issue tracker 的原始请求，给它分类、状态、agent brief 或 needs-info。
  • 试玩、验收或 QA 反馈也可以先写成 issue，再交给 triage 判断状态。

状态词

ready-for-agent 表示 agent 可以无人在旁地开工；ready-for-human 表示需要人类判断或操作；needs-info 表示还缺 reporter 的具体信息。

小测

“PRD 已写好，要拆给多个 agent 并行做。”用哪个 skill？

to-issues。它负责切成可独立抓取、可验收的纵向切片。
