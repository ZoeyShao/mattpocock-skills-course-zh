质量门禁与 Git 安全

第 0009 课 · 工程周期第 6 阶段 · 12 分钟

质量门禁不是最后“看一眼”。它把已经完成的工作放到规范、需求和仓库安全约束下检查。

review：两轴检查

review 从一个固定点比较当前 diff，并分两条轴检查：Standards 看是否符合 repo 规范，Spec 看是否符合 issue/PRD。两轴分开，避免“代码漂亮但做错事”或“需求对但破坏约定”。

自动门禁

  • setup-pre-commit：安装 Husky、lint-staged、Prettier、typecheck/test 钩子。
  • git-guardrails-claude-code：阻止危险 git 命令，比如 push、reset --hard、clean、branch -D。
  • resolving-merge-conflicts：合并/变基冲突时，先理解双方意图，再解决每个 hunk，最后跑检查。

小测

“请 review 这条分支相对 main 的改动。”第一步是什么？

用 review，先固定比较点 main，确认 diff 非空，再找 spec 和 standards 来源。
