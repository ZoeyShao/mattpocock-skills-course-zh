Skill 家族地图

先看这张图。精确选中某个 skill 很重要，但第一步更重要：先选对工作流家族。

家族先问什么Skills
路由与初始化 我们是否需要先寻找、安装或配置 skill 系统？ ask-matt, setup-matt-pocock-skills
诊断与实现 我们是在修 bug、写功能、补测试、做 review，还是解决冲突？ diagnosing-bugs, implement, tdd, review, resolving-merge-conflicts, migrate-to-shoehorn, scaffold-exercises, setup-pre-commit, git-guardrails-claude-code
规划与产品流 我们是否要把模糊想法变成决策、issue、PRD 或 triage 状态？ decision-mapping, to-prd, to-issues, triage
架构与领域语言 我们是否在命名概念、改善模块边界、设计接口，或拉远视角？ codebase-design, domain-modeling, improve-codebase-architecture, prototype
写作与教学 我们是否在整理文章、生成课程，或写更好的 skill？ edit-article, writing-beats, writing-fragments, writing-shape, writing-great-skills, teach
追问、压缩与交接 我们是否需要高强度追问、压缩表达、延续上下文或交接工作？ grilling, grill-me, grill-with-docs, handoff, obsidian-vault
路由规则：先按“最终产物”给任务分类。代码改动、issues、PRD、课程、文章、领域文档和 handoff，通常会指向不同的 skill 家族。

常见缩写

缩写意思什么时候想到它
tdd Test-Driven Development，测试驱动开发。 当你想先写测试、再写实现，并按“红灯、绿灯、重构”的循环推进功能或修 bug。
prd Product Requirements Document，产品需求文档。 当你需要把模糊想法整理成可实现、可验收的需求。
QA Quality Assurance，质量保证或测试反馈。 当你在试玩、验收、报告问题，或把问题整理成 issue。
